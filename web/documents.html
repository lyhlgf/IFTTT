Java 第三次实习报告

高端 刘越晗

1. 实验目的

  • 熟悉Java web 编程技术，包括B/S模式开发、数据库编程、前端开发等
  • 进一步熟悉Java语法， 掌握Java基础类库的使用
  • 体验团队编程，学习任务分配、版本控制等技巧

2. 开发环境

  • 操作系统：分别在Windows 和 Mac osX 上进行开发
  • Java IDE: IntelliJ IDEA
  • Java 版本： 1.8.0
  • Tomcat 版本: 9.0.0
  • 测试使用浏览器: Google Chrome
  • 版本控制： git, 并且使用Github[1]进行云端协作
  • 数据库版本: MySQL
  • 前端模版： Bootstrap[2].

3. 需求整理

3.1 会员功能

3.1.1 登陆注册功能

  • 用户在register界面使用邮箱账号和密码进行注册；

  • 管理员账号后台建立，不可以直接注册。

  • 新建任务

  • 查看、修改、删除任务

任务管理界面可以查看当前用户建立的所有任务，并且可以修改、删除某个任务

  • 运行、暂停任务

和上述功能在同一个界面。

4. 实现细节

4.1 框架流程

4.1.1Sequence diagram

4.1.2项目代码结构

  • lib目录下存放项目用到的jar包；

  • src目录上存放所有的java 源码，包括Servlet和其它公共类；

    • Common子目录
      包含邮件监听、邮件发送、微博监听、微博发送等类。
    • Controller 子目录
    • Login、Register、Init等Servlet；
    • 用户有关的所有Servlet
    • 任务有关的所有Servelt；
    • model 子目录
    • Database类封装数据库访问操作；
    • User类、Task类分别提供用户和任务对象创建、和数据库交互等；
    • weibo4j 子目录
      微博提供的SDK；
    • 1.py文件

      用于根据微博账号和密码自动获取Code；

  • web 目录下存放Bootstrap模版代码以及项目中所有的jsp和html文件；

4.2 前端模版

Bootsrap 是一套快速开发的响应式的前端框架，有丰富的组件和极好的组织方式。

本次实验中使用了Unify、Inspinia两个主题：

  • Login Register页面采用Unify主题
  • 登陆之后的所有界面采用Insinia主题

4.3 主要模块

4.3.1 辅助类

  • ListenWeibo

  • ReadEmail

    负责监听指定邮件有无收到新邮件。

  • SendEmail

    负责使用指定的QQ邮箱发送邮件给某邮箱。
  • SendWeibo

    负责使用账户和密码发送微博。

4.3.2 Controller

  • UserLoginFiter

Task

这里只展示一个Servlet类图，其它完全类似

  • NewTaskServlet

    doPost()负责新建任务的post数据处理和任务建立工作。
    具体的：任务建立不仅包括建立任务，还包括用户余额检测和扣取、消费额和积分额的增长以及消费记录的更新等。
    doGet()负责展示新建任务页面index.jsp

  • TaskManageServlet
    doGet()负责任务显示、修改、删除页面taskMange.jsp的显示
    doPost()负责任务的运行和暂停。

  • TaskDeleteServlet
    doPost()负责任务的删除操作；
  • TaskModifyServelt
    doPost()负责任务的修改操作；
    这里需要根据任务的内容自动调整好修改时对应的this和that事件。

User

  • UserConsumeRecordServlet
    负责用户消费记录的展示。
  • UserIndexServlet
    返回主页面。
  • UserInfoServlet
    负责展示用户信息。
  • UserMailboxServlet
    负责显示用户收到的消息。
  • UserModifyPasswordServlet
    负责修改用户密码并更新数据库。
  • UserRechargeServlet
    负责用户充值。

其它

  • InitServlet
    初始化数据库。
  • LoginServlet
    用户登录，根据数据库内容判断是否合法。
  • LogoutServlet
    用户登出事件处理。
  • RegisterSevlet
    负责用户的注册，以及根据内容生成新的用户。

4.3.3 Model

  • Database

    提供数据库访问的接口：初始化数据库、数据库查询、返回查询结果
  • Consume

    数据库表Consume对应的类。
    负责数据库和用到Consume的程序进行交互。
  • PostMessage

    数据库表PostMessage对应的类。
    负责数据库和用到PostMessage的程序进行交互。
  • Task

    数据库表Task对应的类。
    负责数据库和用到Task的程序进行交互。
  • User

    数据库表User对应的类。
    负责数据库和用到User的程序进行交互。

4.3.1 数据库细节

数据库名称为IFTTT
表名分别为User、Task、Consume、PostMessage

  1.   # 建数据库
  2. create database `IFTTT`;
  4.   # 表User
  5. CREATE TABLE `IFTTT`.`User` (
  6.   `Email` VARCHAR(45) NOT NULL,
  7.   `Password` VARCHAR(45) NOT NULL,
  8.   `Rank` int NOT NULL,
  9.   `Consumption` int ,
  10.   `Balance` int,
  11.   `isAdmin` bit,
  12.    PRIMARY KEY (`Email`));
  14.   # 表Task
  15. CREATE TABLE `IFTTT`.`Task` (
  16.     `taskName` varchar(45) not null,
  17.     `userEmail` varchar(45) not null,
  18.     `timeOrMail` int not null,
  19.     `receiveMail` VARCHAR(45),
  20.     `receiveMailPassword` varchar(45),
  21.     `sendEmail` varchar(45),
  22.     `sendEmailPassword` varchar(45),
  23.     `sendToEmail` varchar(45),
  24.     `date` varchar(45),
  25.     `mailOrWeibo` int not null,
  26.     `weiboAccount` varchar(45),
  27.     `weiboPassword` varchar(45),
  28.     `message` varchar(100),
  29.     `isRunning` bit,
  30.     `ListenWeiBoID` varchar(45),
  31.     `ListenWeiBoPassword` varchar(45),
  32.      `ListenWeiBoMessage` varchar(45),
  33.     PRIMARY KEY (`userEmail`,`taskName`));
  35.   # 表PostMessage
  36. CREATE TABLE `IFTTT`.`PostMessage` (
  37.   `ID` INT NOT NULL,
  38.   `Email` VARCHAR(45) NOT NULL,
  39.   `Subject` VARCHAR(10897) NOT NULL,
  40.   `Content` VARCHAR(10897) NULL,
  41.   `Time` DATETIME NULL,
  42.   `Read` BIT(1) NULL,
  43.   `Important` BIT(1) NULL,
  44.   PRIMARY KEY (`ID`));
  46.   # 表Consume
  47. CREATE TABLE `IFTTT`.`Consume` (
  48.   `ID` INT NOT NULL,
  49.   `ThisDesp` varchar(100) NOT NULL,
  50.   `ThatDesp` varchar(100) NOT NULL,
  51.   `ConsumeValue` int NULL,
  52.   `userEmail` varchar(45) NULL,
  53.   PRIMARY KEY (`ID`));

5. 项目特色

  • 采用Bootstrap响应式前端模版；
  • 项目采用MVC架构组织；
  • 微博的发送和监听增加了自动获取code的方式；
  • 使用Git进行版本控制，Github进行云端协作；
